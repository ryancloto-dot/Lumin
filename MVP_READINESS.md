# Lumin MVP Readiness

This is the product-facing version of the OpenClaw research.

## Must Have

- Real provider execution, not just saved configs
  - First targets: `OpenRouter`, `Ollama`
- Real connector execution, not just saved configs
  - First targets: `Telegram` or `Slack`, plus `Notion`
- OpenClaw migration flow
  - path scan
  - preset import
  - clearer workspace/state/session expectations
- First-run doctor
  - desktop agent
  - provider config
  - connector config
  - NanoClaw root
  - OpenClaw import scan

## Should Have

- Warm NanoClaw runtime reuse
- Better workspace defaults
- Simpler UX vocabulary
- Mobile/dashboard parity on settings and launch-critical flows

## MVP Promise

The MVP should be able to say:

`Connect Telegram or Slack, point Lumin at your repo, choose OpenRouter or Ollama, and control your desktop agent from web or phone in a few minutes.`
